# ADR-0001：发布包按版本目录归档

- 日期：2026-06-25
- 状态：accepted
- 决策人：人工维护者 + Codex

## 背景

`releases/` 目录此前把 Android、Windows、Linux 桌面端和 Linux CLI/Web 的多个版本安装包平铺在同一层。随着版本增加，用户难以判断哪个是最新包，也难以清理旧包。

## 决策

发布包按版本号放置：

```text
releases/
  0.4.0/
  0.4.1/
  0.4.2/
  0.4.3/
  0.5.1/
```

当前版本的所有平台安装包放入 `releases/<version>/`。`RELEASES.md` 记录当前可用包、安装方式和 SHA-256。低于 `0.4.0` 的包不再保留在工作区。

## 后果

- 用户可以直接进入最新版本目录下载。
- 回退版本仍可按版本号查找。
- 发布脚本和文档必须使用 `releases/<version>/...` 路径。
- Git 忽略目录中只保留本地发布产物，不把大二进制包纳入源码管理。
