# StudyShot Relay Agent 开发任务清单

文档版本：2026-06-18（与代码同步核对）  
用途：供后续编码 agent、开发者、测试 agent 按任务推进实现  
关联规格：`study-shot-relay-spec.md`

## 0. 使用说明

本清单不是产品介绍，而是开发执行清单。编码 agent 应先阅读 `study-shot-relay-spec.md`，再按本文任务顺序实现。

实现时必须遵守：

- 不把项目做成网盘或完整相册同步。
- 默认只做“低延迟截图快传”。
- 权限必须落实到设备级。
- 安卓默认只自动监听截图，不默认上传整个相册。
- 上传设备不得自动下载自己上传的图片。
- 安卓下载目录不得再次触发自动上传。
- WebSocket 是实时通知主链路，pending deliveries 是补偿链路。
- 所有下载必须鉴权，不能生成公开图片 URL。
- 不在日志中输出 token、密码、完整敏感路径。
- MVP 不自动控制 GPT 页面，只负责下载和剪贴板。

任务状态建议：

- `[ ]` 未开始。
- `[~]` 进行中。
- `[x]` 完成。
- `[!]` 阻塞。

## 1. 全局 Definition of Done

任意任务完成前，必须满足：

- 代码能构建。
- 新增行为有测试或可手工验证步骤。
- 不破坏已有任务验收。
- 错误状态有明确处理，不静默失败。
- 配置项不硬编码到业务代码。
- 日志不泄露 token、密码、图片内容。
- 文档与实际行为一致。

整个 MVP 完成前，必须通过：

- 平板截图后，Ubuntu 或 Windows 电脑 5 秒内收到图片。
- 电脑自动保存图片。
- 电脑可选自动复制图片到剪贴板。
- 上传设备不会收到自己上传的图片。
- 安卓自动下载的图片不会被再次自动上传。
- 设备撤销后不能继续上传、下载、接收事件。
- 服务器重启后客户端自动恢复连接并补收 pending deliveries。
